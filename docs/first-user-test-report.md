## Цель

Проверить готовность платформы к онбордингу реального эксперта-юриста.

## Сценарий (ручной)

### 1. Клиент

1. `register-client.html` → email `test_client_*@dompro.ru`
2. `client-dashboard.html` → создать заказ (категория «Юриспруденция»)
3. **Ожидание:** заказ в статусе `open`

### 2. Эксперт (Валерия)

1. `register-expert.html` → email `valeria+*@dompro.ru`
2. `expert-dashboard.html` → профиль: ФИО, специализация «Корпоративное право», стаж, описание
3. Верификация: ИНН + документы
4. Админ: `PUT /admin/verifications/{id}/approve`
5. Пополнение кошелька ≥ 150 ₽

### 3. Отклик и accept

1. `workspace.html#feed` или `orders.html` → отклик на заказ
2. Клиент: `order-detail.html?id={id}` → **Принять** отклик
3. **Ожидание:** заказ `in_progress`, отклик `accepted`

### 4. Workspace и договор

1. `workspace.html#clients` → клиент в списке
2. `order-detail.html` → **Создать договор** → PDF скачивается
3. `workspace.html#contracts` → договор в списке

## Автотесты (изолированная БД)

```powershell
cd backend
python scripts/setup_test_db.py
python -m pytest tests/ -v
```

**Ожидание:** все тесты проходят стабильно при повторных запусках.

## Известные ограничения

- Пополнение кошелька — заглушка без эквайринга
- Email welcome-шаблон есть, отправка не подключена
- Фото профиля — только URL, не загрузка файла
- Подписание договора — статус `draft`, без e-sign
